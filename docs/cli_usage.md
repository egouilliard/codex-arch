# Codex-Arch CLI Usage Guide

Codex-Arch provides a comprehensive toolset for analyzing code architecture, extracting dependencies, and visualizing code structure. This guide details all available commands and provides practical examples for common use cases.

## Installation & Setup

Before using the CLI, activate your virtual environment:

```bash
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

## Basic Command Structure

All Codex-Arch commands follow this general pattern:

```bash
codex-arch [command] [arguments] [options]
```

To get help on any command:

```bash
codex-arch --help                # General help with list of all commands
codex-arch [command] --help      # Help for a specific command
```

## Core Commands Reference

### 1. Extracting Dependencies

The `dependencies` command analyzes code to extract dependency relationships between modules and files.

```bash
codex-arch dependencies <repo-path> [options]
```

**Key Options**:
- `--output/-o <dir>`: Output directory for results (default: './output')
- `--file/-f <filename>`: Output filename (default: python_dependencies.json)
- `--include-patterns <patterns>`: Glob patterns for files to include (default: '**/*.py')
- `--exclude-patterns <patterns>`: Glob patterns for files to exclude (default: venv, .git, __pycache__)

**Examples**:
```bash
# Basic dependency extraction
codex-arch dependencies my_project --output analysis_results

# Specify output filename
codex-arch dependencies my_project --output analysis_results --file deps.json

# Exclude specific directories
codex-arch dependencies my_project --exclude-patterns "**/tests/**" "**/docs/**"
```

### 2. Visualizing Dependencies

There are two recommended approaches for visualizing dependencies:

#### 2.1 Basic Architecture Visualization

For standard architecture visualization, use the `generate_arch_graph.py` script:

```bash
# 1. Generate dependency data
codex-arch dependencies my_project --output output

# 2. Generate architecture visualization
python generate_arch_graph.py output/python_dependencies.json output/architecture_graph
```

This script generates both SVG and PNG versions of the architecture diagram with:
- Module-level dependencies
- Submodule relationships
- Dependency counts between modules
- Highlighted core components
- Visual grouping of related code

**Command Syntax**:
```bash
python generate_arch_graph.py <input-dependency-json-file> <output-file-path>
```

**Parameters**:
- `<input-dependency-json-file>`: Path to the JSON file containing dependency data (typically generated by `codex-arch dependencies`)
- `<output-file-path>`: Path for the output files (without extension, as both .svg and .png will be generated)

**Example**:
```bash
python generate_arch_graph.py output/python_dependencies.json output/architecture_graph
```

The script will generate:
- `output/architecture_graph.svg`: Vector format suitable for documentation
- `output/architecture_graph.png`: Raster format suitable for presentations

#### 2.2 Enhanced Architecture Visualization

For more detailed architecture diagrams, first convert the dependency format:

```bash
# 1. Generate dependency data
codex-arch dependencies my_project --output output

# 2. Convert the dependency format
python convert_deps.py output/python_dependencies.json output/complete_dependencies.json

# 3. Generate enhanced architecture diagram
python generate_arch_graph.py output/complete_dependencies.json output/complete_arch_graph
```

This approach creates an optimized dependency graph with better organization of modules and components.

**Customizing the Architecture Diagram**:

The `generate_arch_graph.py` script can be modified to:
- Highlight different components by editing the `core_files` list
- Change the visual style by modifying the GraphViz attributes
- Focus on specific modules by editing the pattern matching logic

### 3. Generating File Trees

The `filetree` command creates a hierarchical representation of a directory structure.

```bash
codex-arch filetree <directory-path> [options]
```

**Key Options**:
- `--output/-o <file>`: Output file path (default: stdout)
- `--format/-f {json,markdown,md}`: Output format
- `--max-depth/-d <depth>`: Maximum depth to traverse
- `--exclude-dirs <dirs>`: Directories to exclude (e.g., .git, node_modules)
- `--exclude-patterns <patterns>`: Regex patterns to exclude files and directories
- `--include-hidden`: Include hidden files and directories
- `--follow-symlinks`: Follow symbolic links
- `--no-metadata`: Exclude metadata from JSON output
- `--relative-paths`: Use paths relative to the root path

**Examples**:
```bash
# Generate JSON file tree
codex-arch filetree my_project --output project_structure.json

# Generate markdown file tree
codex-arch filetree my_project --output project_structure.md --format markdown

# Limit tree depth
codex-arch filetree my_project --output project_structure.json --max-depth 3

# Exclude specific directories
codex-arch filetree my_project --output project_structure.json --exclude-dirs venv node_modules .git
```

### 4. Collecting Code Metrics

The `metrics` command analyzes code files to collect metrics like line count, complexity, and more.

```bash
codex-arch metrics <directory-path> [options]
```

**Key Options**:
- `--output/-o <file>`: Output file path (default: './output/metrics.json')
- `--exclude-dirs <dirs>`: Directories to exclude
- `--exclude-patterns <patterns>`: Regex patterns to exclude files
- `--exclude-extensions <exts>`: File extensions to exclude
- `--include-extensions <exts>`: Only include these file extensions
- `--include-hidden`: Include hidden files and directories
- `--max-file-size <size>`: Maximum file size in bytes to process (default: 10MB)
- `--no-complexity`: Skip complexity analysis
- `--complexity-max-file-size <size>`: Maximum file size for complexity analysis (default: 1MB)

**Examples**:
```bash
# Basic metrics collection
codex-arch metrics my_project --output project_metrics.json

# Skip complexity analysis for faster processing
codex-arch metrics my_project --output project_metrics.json --no-complexity

# Focus on specific file types
codex-arch metrics my_project --output project_metrics.json --include-extensions .py .js
```

### 5. Full Analysis Pipeline

The `analyze` command runs a complete analysis pipeline including file tree, dependencies, metrics, and visualization.

```bash
codex-arch analyze <repo-path> [options]
```

**Key Options**:
- `--output/-o <dir>`: Output directory for all results (default: './output')
- `--skip-file-tree`: Skip file tree analysis
- `--skip-dependencies`: Skip dependency analysis
- `--skip-metrics`: Skip metrics collection
- `--skip-visualization`: Skip visualization generation
- `--skip-summary`: Skip summary generation
- `--bundle`: Create a context bundle with all artifacts
- `--exclude-dirs <dirs>`: Directories to exclude from all analyses

**Examples**:
```bash
# Run complete analysis
codex-arch analyze my_project --output full_analysis

# Skip visualization generation
codex-arch analyze my_project --output full_analysis --skip-visualization

# Focus on specific analyses
codex-arch analyze my_project --output full_analysis --skip-metrics --skip-summary

# Exclude common directories
codex-arch analyze my_project --output full_analysis --exclude-dirs venv .git node_modules dist
```

### 6. Complete End-to-End Analysis

The `run-all` command provides a comprehensive way to run all analysis steps in one go, including visualization generation:

```bash
codex-arch run-all <repo-path> [options]
```

**Key Options**:
- `--output/-o <dir>`: Output directory for all results (default: './output')
- `--exclude-dirs/-d <dirs>`: Directories to exclude from all analyses
- `--exclude-patterns/-e <patterns>`: Regex patterns to exclude files and directories
- `--include-hidden`: Include hidden files and directories
- `--convert-deps`: Convert dependency format for enhanced visualization
- `--no-complexity`: Skip complexity analysis for faster processing
- `--no-metrics`: Skip metrics collection
- `--no-file-tree`: Skip file tree generation
- `--no-visualization`: Skip architecture visualization

**Examples**:
```bash
# Run all analysis steps with standard visualization
codex-arch run-all my_project --output analysis_results --exclude-dirs venv .git node_modules

# Run all analysis steps with enhanced visualization
codex-arch run-all my_project --output analysis_results --exclude-dirs venv .git node_modules --convert-deps

# Run analysis with specific options
codex-arch run-all my_project --output analysis_results --no-complexity --convert-deps
```

This command performs:
1. Full code analysis (dependencies, metrics, file tree)
2. Dependency format conversion (if --convert-deps is specified)
3. Standard architecture graph generation 
4. Enhanced architecture graph generation (if --convert-deps is specified)

The `run-all` command provides the easiest path to generating a complete analysis with visualizations in one step, without having to run multiple commands manually.

### 7. Creating Summary Reports

The `summary` command generates a comprehensive summary report of the codebase architecture.

```bash
codex-arch summary <directory-path> [options]
```

**Key Options**:
- `--output/-o <file>`: Output file path (default: './output/summary.md')
- `--template {standard,detailed,minimal}`: Summary template to use
- `--exclude-dirs <dirs>`: Directories to exclude
- `--include-metrics`: Include metrics in the summary (default: True)
- `--include-dependencies`: Include dependency analysis in the summary (default: True)
- `--include-visualizations`: Include visualizations in the summary (default: True)

**Examples**:
```bash
# Generate standard summary
codex-arch summary my_project --output project_summary.md

# Generate detailed summary
codex-arch summary my_project --output project_summary.md --template detailed

# Generate minimal summary
codex-arch summary my_project --output project_summary.md --template minimal
```

### 8. Context Bundle Creation

The `bundle` command packages all analysis artifacts into a structured bundle for consumption by LLMs or other tools.

```bash
codex-arch bundle <repo-path> [options]
```

**Key Options**:
- `--output/-o <dir>`: Output directory for results (default: './output')
- `--bundle-dir <dir>`: Directory name for the bundle within the output directory
- `--no-file-tree`: Don't include file tree artifacts
- `--no-dependencies`: Don't include dependency artifacts
- `--no-metrics`: Don't include metrics artifacts
- `--no-visualizations`: Don't include visualization artifacts
- `--no-summaries`: Don't include summary artifacts
- `--compress`: Compress the bundle into a single file
- `--compression-format {zip,tar,tar.gz}`: Format for compression

**Examples**:
```bash
# Create a basic bundle
codex-arch bundle my_project --output analysis_bundle

# Create a compressed bundle
codex-arch bundle my_project --output analysis_bundle --compress

# Create a bundle with only dependencies and metrics
codex-arch bundle my_project --output analysis_bundle --no-file-tree --no-visualizations --no-summaries
```

### 9. API Service

The `api` command launches a REST API server providing access to Codex-Arch functionality.

```bash
codex-arch api [options]
```

**Key Options**:
- `--host <host>`: Host to bind the server to (default: 0.0.0.0)
- `--port <port>`: Port to bind the server to (default: 5000)
- `--debug`: Run the server in debug mode
- `--env {dev,test,prod}`: Environment to run the server in (default: dev)
- `--output-dir <dir>`: Base directory for analysis output files
- `--cors-origins <origins>`: CORS allowed origins (comma-separated)

**Examples**:
```bash
# Start API server with default settings
codex-arch api

# Start API server on a specific port
codex-arch api --port 8080

# Start API server in production mode
codex-arch api --env prod --port 8080
```

## Common Workflow Examples

### Basic Code Analysis

```bash
# Activate virtual environment
source venv/bin/activate

# Generate dependency graph
codex-arch dependencies my_project --output output

# Generate architecture visualization
python generate_arch_graph.py output/python_dependencies.json output/architecture_graph

# Collect code metrics
codex-arch metrics my_project --output output/metrics.json
```

### Enhanced Architecture Visualization

```bash
# 1. Extract dependencies
codex-arch dependencies my_project --output output

# 2. Convert dependency format for better visualization
python convert_deps.py output/python_dependencies.json output/complete_dependencies.json

# 3. Generate the enhanced architecture diagram
python generate_arch_graph.py output/complete_dependencies.json output/complete_arch_graph

# 4. View the resulting SVG or PNG file in a browser or image viewer
```

### Complete Analysis with a Single Command

```bash
# Run all analysis steps with a single command
codex-arch run-all my_project --output analysis_results --exclude-dirs venv .git node_modules --convert-deps
```

This provides the most efficient way to generate a complete analysis with visualizations in one step.

### Focused Analysis on Specific Directories

```bash
# Analyze only the src directory
codex-arch dependencies my_project/src --output output

# Generate architecture diagram
python generate_arch_graph.py output/python_dependencies.json output/architecture_graph
```

### Generate Documentation Bundle

```bash
# Generate complete analysis
codex-arch analyze my_project --output output

# Generate architecture visualization for the bundle
python generate_arch_graph.py output/python_dependencies.json output/architecture_graph

# Create a bundled package for documentation
codex-arch bundle my_project --output docs_bundle --compress
```

## Working with Large Codebases

For large codebases, consider the following optimization strategies:

1. **Target specific directories**:
   ```bash
   codex-arch dependencies my_project/src --output analysis
   ```

2. **Exclude non-code directories**:
   ```bash
   codex-arch analyze my_project --exclude-dirs venv node_modules .git dist tests docs examples
   ```

3. **Use file pattern filtering**:
   ```bash
   codex-arch dependencies my_project --include-patterns "**/*.py" "**/*.js" --exclude-patterns "**/test_*.py"
   ```

4. **Limit visualization complexity**:
   ```bash
   python generate_arch_graph.py analysis/python_dependencies.json analysis/architecture_graph
   ```

5. **Skip complex analyses when not needed**:
   ```bash
   codex-arch metrics my_project --no-complexity
   ```

## Known Issues & Troubleshooting

1. **Summary generation errors**: The summary command may encounter errors in some versions.
   - Workaround: Use individual commands for analysis and create your own summary.

2. **Bundle creation errors**: The bundle command may encounter errors in some versions.
   - Workaround: Manually collect the output files from the analysis commands.

3. **Performance with large codebases**: Very large codebases may cause memory issues.
   - Solution: Focus analysis on specific directories or use file pattern filtering.

4. **Built-in visualization command issues**: The built-in `visualize` command may produce empty graphs.
   - Solution: Use the `generate_arch_graph.py` script directly as shown in the examples above.

## Environment Variables

- `CODEX_ARCH_LOG_LEVEL`: Set default logging level
- `CODEX_ARCH_OUTPUT_DIR`: Default output directory
- `CODEX_ARCH_DEFAULT_EXCLUDES`: Default directories to exclude (comma-separated)

## Tips for Best Results

1. **Start small**: For new codebases, start with individual directory analysis before running full analysis
2. **Use specific commands**: For targeted insights, prefer individual commands over the full `analyze` pipeline
3. **Check visualization formats**: SVG output offers better interactivity than PNG
4. **Convert dependency data**: For complex codebases, use the convert_deps.py script before visualization
5. **Review metrics first**: Understanding code metrics can help prioritize which parts of the codebase need deeper analysis
